# 판례암기 (법무사 2차)

빌드 도구 없는 순수 HTML/CSS/JS PWA. 판례를 간격반복(SRS) 방식으로 암기하는 모바일 학습 게임입니다.

## 실행 방법

서비스워커(오프라인 캐싱)는 `file://`로 직접 열면 등록되지 않으므로, 반드시 정적 서버로 서빙해야 합니다.

```bash
cd pansure-quiz
python -m http.server 8000
```

그 다음 브라우저에서 `http://localhost:8000` 접속.

### 휴대폰에서 테스트하기

1. PC와 휴대폰이 같은 Wi-Fi에 연결되어 있어야 합니다.
2. PC의 로컬 IP를 확인합니다 (Windows: `ipconfig`에서 IPv4 주소 확인).
3. 휴대폰 브라우저에서 `http://<PC의 IP>:8000` 접속.
4. 크롬(안드로이드)은 주소창 메뉴 → "홈 화면에 추가", 사파리(아이폰)는 공유 버튼 → "홈 화면에 추가"로 앱처럼 설치할 수 있습니다.

## 콘텐츠 수정/확장

- `js/data.js`에 판례 카드가 들어있습니다. 새 카드를 추가하려면 기존 항목과 같은 형식(`id`, `subject`, `caseNumber`, `date`, `issue`, `holding`, `keyword`, `articles`)으로 배열에 객체를 추가하면 됩니다.
- **⚠️ 초기 데이터의 사건번호/선고일은 조사 기반 초안이며 실전 사용 전 최신 판례집과 반드시 대조 확인하세요.** (`js/data.js` 상단 주석 참고)
- `keyword`는 반드시 `holding` 문장에 그대로 포함된 문자열이어야 빈칸 문제가 정상 동작합니다.

## 구조

- `index.html` / `styles.css` — 앱 셸, 모바일 퍼스트 스타일
- `js/data.js` — 판례 데이터셋
- `js/srs.js` — 간격반복 스케줄링 (SM-2 간소화)
- `js/storage.js` — localStorage 저장/불러오기
- `js/app.js` — 화면 전환 및 퀴즈 로직
- `manifest.json` / `sw.js` / `icons/` — PWA 설정 (홈 화면 설치, 오프라인 캐싱)

## 학습 흐름

1. 홈 화면에서 기본/심화 탭과 과목별 오늘 복습 예정 카드 수 확인 후 "학습 시작"
2. 문제(OX / 빈칸 4지선다 / 사례형 4지선다 중 랜덤)에 답하면 정답과 판례 취지가 공개됨
3. "다시 / 어려움 / 좋음 / 쉬움" 중 체감 난이도를 선택하면 다음 복습 시점이 자동으로 조정됨 (SRS)
4. 통계 화면에서 연속 학습일과 최근 7일 활동을 확인 가능

## 업데이트가 화면에 안 보일 때 (캐시 문제)

이 앱은 오프라인 사용을 위해 서비스워커(`sw.js`)로 파일을 캐싱합니다. `sw.js`는 네트워크 우선(network-first) 전략이라 온라인 상태면 항상 최신 파일을 받아오지만, 이미 이전 버전(캐시 우선 방식)으로 한 번 방문한 기기는 옛 서비스워커가 남아있어 새로고침만으로는 갱신되지 않을 수 있습니다. 이 경우:

- **모바일(홈 화면 아이콘)**: 브라우저 설정 → 이 사이트의 데이터/캐시 삭제 후 다시 접속하거나, 홈 화면 아이콘을 삭제하고 다시 추가
- **데스크톱 브라우저**: 개발자도구 → Application(또는 저장공간) 탭 → Service Workers에서 Unregister 후 새로고침
- 이후로는 `sw.js`가 네트워크 우선 방식이라 이런 문제가 재발하지 않습니다.
